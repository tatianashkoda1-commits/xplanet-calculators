# Калькуляторы Планеты Х

Серия из 5 standalone HTML-калькуляторов для встраивания на сайт **xplanet.club**.

**Live превью:** https://tatianashkoda1-commits.github.io/xplanet-calculators/

## Что внутри

| Файл | Калькулятор |
|---|---|
| `index.html` | Хаб с карточками-ссылками на все 5 |
| `portions.html` | Пересчёт порций (по числу или от ингредиента) |
| `spoons.html` | Граммы ↔ ложки/стаканы для 24 продуктов |
| `oven.html` | Температуры духовки: °C ↔ °F ↔ газовая отметка |
| `form.html` | Расход на форму (по площади, любые формы) |
| `cost.html` | Себестоимость порции, наценка, цена продажи |
| `sample-prices.json` | Пример словаря цен на 46 продуктов (для тестов `cost.html`) |

## Как встраивать

Каждый файл самодостаточный: HTML + CSS + JS внутри одного файла, без сборки и зависимостей кроме Google Fonts. Два пути:

1. **Как отдельные страницы** — положить файлы в раздел сайта, давать на них ссылки из меню/хаба
2. **Как блоки внутри ваших страниц** — вытащить из каждого `<style>`, `<script>` и разметку основного блока, вставить в существующий шаблон

## Что важно учесть при интеграции

### `cost.html` — личный список цен

Калькулятор себестоимости имеет «свой словарь продуктов» — пользователь сохраняет цены, в следующих рецептах они подтягиваются автоматом.

В демо-версии словарь хранится в `localStorage` браузера. **На проде** нужно подменить на запросы к API личного кабинета xplanet.club. Точка подмены в начале JS-блока `cost.html` помечена комментарием:

```
// === ХРАНЕНИЕ ЦЕН — ЗОНА ИНТЕГРАЦИИ С ЛИЧНЫМ КАБИНЕТОМ XPLANET.CLUB ===
```

Там же — структура объекта и список из 4 функций, которые надо превратить в API-вызовы.

### Канон стиля

- Палитра: cream `#FDF8F3`, terracotta `#E85D3A`, gold `#D4A052`, dark `#2D2A26`
- Шрифты: Inter (основной) + Caveat (рукописный, для акцентов)
- Скруглённые карточки, hover с лёгким подъёмом, кнопки-pill
- Адаптив: одна колонка на ширине ≤ 880–980 px

Если на сайте уже есть собственная палитра/шрифты — лучше адаптировать под неё, чем оставлять дублирующие токены.

## Технические грабли (важно)

1. **Поля чисел:** используется `type="text" inputmode="decimal"`, а не `type="number"`. Причина — `type="number"` не принимает значения с запятой, а русский пользователь пишет «1,5», не «1.5».
2. **Парсинг ингредиентов работает с кириллицей.** В JS regex `\b` и `\W` не понимают русских букв — внутри используется `\p{L}+/gu` и lookahead на пунктуацию.
3. **GitHub Pages + кириллические имена файлов = 404.** Поэтому имена латинские. Если будете деплоить как поддомен/раздел — сохраните это правило.
